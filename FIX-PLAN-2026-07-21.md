Orchestrated by Fable 5; implemented by Opus 4.8 subagents on disjoint files; reviewed and finalized by Fable 5.

## Bug 1 (HIGH) — Duel reward duplication on reload
Root cause: duels never mark a payout as credited server-side (the teacher-question flow does, coop-server.js ~L228). `scoreDuel` writes `d.result.delta`; `duelView` re-emits it for all of `DUEL_DONE_MS` (60s); the client's `applied{}` guard (coop-client.js ~L611) is in-memory and resets on reload.
Fix (Agent A, `coop/coop-server.js` + `coop/coop-client.js`):
- Server: record a per-player `credited` set on the duel; when `duelView` reports the result to a player, include their delta only the first time, then zero it on subsequent reports (mirror `tqStudentView`). Keep the result visible (win/lose text) but non-re-crediting.
- Client: keep `applied{}` but rely on the server delta being one-shot; ensure a reloaded winner reading a stale done-duel gets delta 0.

## Bug 2 (HIGH) — Nation-name stored XSS
